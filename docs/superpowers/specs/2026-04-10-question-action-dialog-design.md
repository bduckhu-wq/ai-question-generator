# 题目操作弹窗重设计

## 背景

当前题目操作使用一个 480px 的 `el-dialog` 弹窗，内部平铺了题目预览、快捷调整（换难度/换题型/删除）、自定义改编、分值调整四个区域。存在以下问题：

1. **内容分区不合理**：预览区域占用大量空间，但老师在试卷上已经能看到题目内容
2. **操作逻辑不完善**：换难度/换题型采用机械轮换（中等→困难），不符合老师真实使用场景
3. **信息层级不清晰**：所有操作平铺堆叠，没有主次关系

## 设计目标

- 弹窗聚焦「换题和调整」，操作即时生效
- 换难度/换题型支持展开选择，而非机械轮换
- 新增「换一道类似的」高频操作
- 删除弱化到标题栏
- 自定义改编始终可见，引导老师用自然语言描述需求

## 弹窗结构


## 各操作项交互细节

### 1. 精简摘要区

弹窗顶部显示题目元信息摘要，不展示完整题目内容：

- 第一行：题型标签 + 难度标签 + 分值（如 `选择题 · 中等 · 5分`）
- 第二行：知识点标签（如 `知识点：二次函数`，仅当题目有知识点信息时显示）
- 使用 `el-tag` 组件，不同类型用不同颜色区分

### 2. 换难度

- 默认折叠，按钮显示当前难度值（如 `换难度  中等 ▾`）
- 点击后在按钮下方展开难度选项列表（简单 / 中等 / 困难）
- 当前难度项高亮标记（radio 样式）
- 选择后即时替换题目，弹窗保持打开
- 题库无匹配时显示提示「该难度下暂无同类题目」
- 再次点击按钮或选择后自动折叠

### 3. 换题型

- 默认折叠，按钮显示当前题型（如 `换题型  选择题 ▾`）
- 点击后展开题型选项列表，列表根据当前学科动态显示（通过 `SUBJECT_QUESTION_TYPES` 获取）
- 当前题型高亮标记
- 选择后即时替换题目，弹窗保持打开
- 无匹配时提示「该题型下暂无同类题目」

### 4. 换一道类似的

- 单按钮，一键执行，不需要展开选项
- 保持当前难度 + 题型 + 知识点不变，从题库/AI 换一道同属性题目
- 点击后按钮显示 loading 状态（旋转图标），替换完成后恢复
- 可连续点击多次换题
- 这是老师最高频的操作，放在操作区第二行首位

### 5. 分值调整

- 紧凑的数字步进器，内联显示（如 `分值  [- 5 +]`）
- 使用 `el-input-number`，`controls-position="right"`，尺寸 `small`
- 改变数值后即时生效（300ms 防抖）
- 范围 1-20 分，步进 1
- 去掉原来的「应用」按钮

### 6. 自定义改编

- textarea 始终可见，默认 2 行高度
- 聚焦时自动扩展到 4 行（CSS transition）
- placeholder：`请描述修改需求，例如：换成带图形的几何题、降低计算量`
- 下方显示引导提示（💡 图标 + 示例文案）
- 输入内容后按回车或点击发送按钮提交
- 提交后弹窗关闭，AI 开始改编流程
- textarea 为空时发送按钮禁用

### 7. 删除（弱化入口）

- 位置：标题栏右侧，关闭按钮左边
- 样式：灰色小文字「删除」，hover 时文字变红
- 点击后不弹新 dialog，而是在删除按钮下方显示一个轻量确认气泡（popover）：
  - 内容：「确定删除此题？」
  - 按钮：「取消」「确定删除」（红色）
- 确认后删除题目并关闭操作弹窗
- 取消后气泡消失，弹窗保持打开

## 弹窗行为规则

- **即时生效**：换难度、换题型、换类似、分值调整均为即时生效，不需要统一确认
- **弹窗不自动关闭**：除删除和自定义改编提交外，其他操作完成后弹窗保持打开，方便继续调整
- **摘要实时更新**：操作生效后，顶部摘要区的标签实时更新（如难度从「中等」变为「困难」）
- **关闭方式**：点击关闭按钮、点击遮罩层、按 Esc 键均可关闭
- **宽度**：保持 480px，高度自适应内容
